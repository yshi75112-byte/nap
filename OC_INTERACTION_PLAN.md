# OC Friendly Map Interaction Plan

## 目标定位

把地图编辑器做成 OC 世界观创作工具，而不是专业 GIS 或纯绘图软件。

核心体验是：

1. 用户先画一个地方。
2. 系统引导用户命名、选择类型、建立归属。
3. 每个地域天然带有叙事信息、角色关系和层级结构。

一句话目标：

> 从「画范围 -> 选类型 -> 命名 -> 绑定」简化为「画 -> 命名 -> 归属」。

## 核心原则

1. 从下往上创建：默认从城市、家族、据点等小地域开始，再向上归属到国家或大陆。
2. 三步完成：画、命名、归属，不打断创作流。
3. 叙事优先：地图不是静态图片，而是世界观入口。
4. 自动建议，不强制决定：系统可以自动检测归属，但必须允许用户改。
5. 地域和图层分离：地域表达世界结构，图层表达绘图结构。

## 信息架构

### Region 地域

地域用于表示世界观中的地理或势力单位。

字段建议：

```ts
type RegionType = "city" | "family" | "country" | "continent";

type Region = {
  id: string;
  name: string;
  type: RegionType;
  parentId?: string;
  geometry: unknown;
  color: string;
  summary?: string;
  characterIds: string[];
  createdAt: string;
  updatedAt: string;
};
```

### Layer 图层

图层用于表示绘图对象的组织方式。

字段建议：

```ts
type LayerType = "region" | "decoration" | "route" | "note" | "draft";

type Layer = {
  id: string;
  name: string;
  type: LayerType;
  regionId?: string;
  visible: boolean;
  locked: boolean;
  order: number;
  createdAt: string;
  updatedAt: string;
};
```

## 阶段一：MVP 主流程

目标：验证「画 -> 命名 -> 归属」是否足够顺。

### 1. 默认进入绘制模式

进入页面后，画布默认可绘制。

功能：

- 默认工具为柳叶笔。
- 默认色板选择第一个颜色。
- 用户在画布上绘制封闭区域。
- 松手后自动闭合路径。
- 自动使用当前色板颜色填充地域。

验收标准：

- 用户不需要先点工具就能开始画。
- 绘制后画布上出现一个封闭地域。
- 地域边界和填充色清晰可见。

### 2. 绘制后弹出轻量命名弹窗

功能：

- 名称必填。
- 类型默认选中「城市」。
- 简介选填。
- 回车可直接完成。
- 取消时删除刚刚绘制的临时地域。

验收标准：

- 用户画完后不用找按钮，弹窗自动出现。
- 只输入名称并回车即可完成创建。
- 没有名称时不能提交。

### 3. 自动建议归属

功能：

- 系统检测新地域是否位于已有地域范围内。
- 如果命中已有地域，自动建议归属。
- 如果没有命中，提示用户选择上级地域或新建上级地域。
- 用户可以更改自动建议。

建议文案：

- 命中已有地域：`已归属到「国家 A」`
- 未命中已有地域：`这个城市属于哪个国家？`

验收标准：

- 在国家内画城市时，系统能建议该国家作为上级。
- 用户可以接受建议，也可以更改归属。
- 没有上级地域时，用户可以暂时跳过或新建。

### 4. 地域标签显示

功能：

- 地图模式下显示地域名称。
- 名称跟随地域中心点展示。
- 当前视图只显示当前层级相关地域。

验收标准：

- 用户能看清每个地域名称。
- 名称不会明显遮挡边界。
- 双击地域前，用户能判断自己点的是哪个地域。

## 阶段二：地图查看与叙事

目标：让地图成为 OC 世界观浏览入口。

### 1. 地图模式

功能：

- 显示所有当前层级地域。
- 显示地域名称和边界线。
- 悬停地域时高亮边界。
- 悬停后显示故事卡片。

故事卡片内容：

- 名称
- 类型
- 简介
- 关联角色列表
- 编辑按钮

验收标准：

- 悬停地域时反馈明显。
- 故事卡片不遮挡鼠标附近的主要边界。
- 没有简介或角色时，卡片仍然自然显示。

### 2. 双击进入下级视图

功能：

- 双击地域进入该地域的子地域视图。
- 左上角显示面包屑。
- 面包屑支持点击返回上级。
- 当前视图背景淡化显示上级地域轮廓。

面包屑示例：

```text
世界 > 大陆 A > 国家 B
```

验收标准：

- 双击国家后，只重点显示该国家下的城市、家族或据点。
- 用户能通过面包屑返回上级。
- 进入下级后不会失去空间方向感。

## 阶段三：编辑模式与基础图层

目标：支持用户修改已有地域，并提供够用的图层管理。

### 1. 编辑模式入口

入口：

- 顶部工具栏「编辑」按钮。
- 故事卡片里的「编辑」按钮。
- 双击进入地域后点击「编辑」。

功能：

- 编辑模式下悬停地域时边线闪烁。
- 地域区域轻微抬起。
- 单击地域后进入绘制或修改状态。

验收标准：

- 用户能明确区分查看模式和编辑模式。
- 悬停反馈保留原有设计中的闪烁和微抬起效果。

### 2. 绘制绑定地域

功能：

- 在某个地域下绘制时，新内容默认绑定到当前地域。
- 内容可以超出上级边界显示。
- 不使用剪切蒙版强行裁切。

验收标准：

- 用户能在城市周围画路径、标注或装饰。
- 超出地域边界的内容仍可见。

### 3. 图层面板 MVP

位置：

- 右下角。
- 默认收起。
- 点击展开。

第一版功能：

- 显示图层列表。
- 新建空白图层。
- 显示 / 隐藏图层。
- 锁定 / 解锁图层。
- 重命名图层。
- 删除图层。
- 调整图层顺序。

暂缓功能：

- 合并图层。
- 图层分组。
- 图层混合模式。

验收标准：

- 每个地域默认创建一个绑定图层。
- 用户能创建装饰或标注图层。
- 锁定图层后不能误编辑。

## 阶段四：颜色与风格

目标：让色彩体系更像世界观风格，而不是普通画板。

### 1. 默认风格预设

建议预设：

- 羊皮纸幻想：羊皮纸底、墨绿、暗金、酒红、深蓝。
- 东方古风：朱砂、黛蓝、牙白、墨黑。
- 霓虹赛博：霓虹紫、电光蓝、荧光粉、暗灰。
- 雪国寒境：冰蓝、银白、冷灰、深青。
- 荒原废土：锈红、沙黄、焦黑、军绿。

验收标准：

- 用户能一键切换整套色板。
- 当前选中色槽有明确状态。
- 新绘制地域使用当前颜色。

### 2. 吸管工具

功能：

- 点击吸管图标进入取色状态。
- 点击画布上的任意地域后取色。
- 取到的颜色写入当前选中色槽。
- 支持跨层级取色。

验收标准：

- 用户能从已有地域吸取颜色。
- 吸色后自动回到绘制工具或上一个工具。

### 3. 自定义色板

建议作为 Pro 功能。

功能：

- 保存自定义色板。
- 重命名色板。
- 删除色板。
- 设置默认色板。

验收标准：

- 免费用户可以使用默认预设。
- Pro 用户可以保存自己的世界观色板。

## 阶段五：导出与分享

目标：让创作者能把地图发出去，或继续二次编辑。

### 1. 导出

免费导出：

- PNG 白色背景。
- 带水印 PNG。

Pro 导出：

- PNG 透明背景。
- 无水印 PNG。
- SVG。

验收标准：

- 导出的图片包含当前视图层级。
- 地域名称和边界清晰。
- 水印不遮挡地图主体。

### 2. 分享链接

功能：

- 生成只读分享链接。
- 链接包含地图名称。
- 链接包含创建者信息。
- 链接打开后进入当前视图层级。
- 访问者只能查看，不能编辑。

验收标准：

- 分享者看到的是当前层级。
- 被分享者不能修改地图数据。
- 故事卡片在只读模式下仍可查看。

## 推荐开发顺序

### 第 1 周：核心绘制闭环

1. 搭建画布基础。
2. 实现柳叶笔绘制封闭区域。
3. 绘制后创建临时地域。
4. 弹出命名弹窗。
5. 保存 Region 数据。
6. 在画布上显示名称和边界。

完成标志：

> 用户可以画一个城市、命名它，并在地图上看到它。

### 第 2 周：归属与层级

1. 实现 Region parentId。
2. 实现地域包含检测。
3. 实现自动建议归属。
4. 实现手动选择归属。
5. 实现双击进入下级视图。
6. 实现面包屑返回。

完成标志：

> 用户可以先画城市，再把它归到国家里，并进入国家视图查看子地域。

### 第 3 周：故事卡片

1. 实现悬停检测。
2. 实现地域高亮。
3. 实现故事卡片。
4. 支持简介编辑。
5. 支持关联角色列表展示。
6. 支持从故事卡片进入编辑。

完成标志：

> 用户悬停一个地域时，可以看到它的设定和相关角色。

### 第 4 周：编辑模式与图层 MVP

1. 增加查看 / 编辑模式切换。
2. 保留悬停边线闪烁和区域微抬起效果。
3. 实现图层面板。
4. 实现图层显示 / 隐藏。
5. 实现图层锁定。
6. 实现新建、重命名、删除、排序。

完成标志：

> 用户可以管理地域图层和装饰图层，不容易误删或误改。

### 第 5 周：颜色风格

1. 实现默认风格预设。
2. 实现色槽状态。
3. 实现吸管工具。
4. 实现跨层级取色。
5. 预留自定义色板接口。

完成标志：

> 用户可以用不同世界风格快速画出有氛围的地图。

### 第 6 周：导出与分享

1. 实现 PNG 导出。
2. 实现白底 / 透明背景选项。
3. 实现水印策略。
4. 实现只读分享链接。
5. 分享链接保留当前视图层级。

完成标志：

> 用户可以把地图导出或分享给别人查看。

## 关键交互细节

### 创建地域最短路径

```text
打开页面 -> 直接画 -> 输入名称 -> 回车 -> 自动建议归属 -> 完成
```

目标耗时：

```text
10 秒内完成一个城市创建
```

### 归属提示策略

如果检测到上级地域：

```text
已归属到「国家 A」  更改
```

如果没有检测到上级地域：

```text
这个城市属于哪个国家？
选择已有国家 / 新建国家 / 暂时跳过
```

### 故事卡片空状态

没有简介时：

```text
还没有简介
```

没有角色时：

```text
暂无关联角色
```

不要显示过重的引导文案，保持轻。

## 风险与注意点

1. 自动闭合路径要足够宽容，不能要求用户画得很准。
2. 地域包含检测可能有复杂边界，需要先接受近似判断。
3. 双击进入下级视图后，要避免用户迷路。
4. 图层和地域必须分离，否则后期功能会互相牵制。
5. 故事卡片不能做成笨重表单，应该保持轻量。
6. 导出 SVG 会暴露数据结构复杂度，可以等 PNG 稳定后再做。

## 第一版不做的事

- 不做复杂 GIS 坐标系统。
- 不做真实地理比例尺。
- 不做多人实时协作。
- 不做完整图像编辑器。
- 不做复杂蒙版和混合模式。
- 不强制用户先建大陆或国家。

## 下一步

建议先进入第 1 周任务：

1. 确认当前项目技术栈。
2. 找到画布或地图编辑相关入口。
3. 建立 Region / Layer 的基础数据模型。
4. 实现最短创建闭环。
